\hh{Управление сервером}\n\h{Первичная настройка}
\n\tПри первом запуске сервера начинается процедура первичной настройки.
Пользователю будет предложено выбрать для сервера узла имя и UUID.
Имя имеет преимущественно декоративный характер (используется для обозначения узла в интерфейсах различных программ).
Имя может использоваться для адресации узла в программе командной строки ecfremc.
UUID - уникальный идентификатор узла (128 бит). Все узлы должны иметь различные UUID.
Один узел может быть частью различных кластеров. Его адреса в рамках кластеров могут различаться.
UUID же является постоянным. Система автоматически предлагает пользователю случайно сгенерированный UUID,
рекомендуется использовать его.
\n\h{Настройка кластера}
\n\tПанель управления кластером открывается при щелчке на его значке в панеле состояния системы.
Основные кнопки управления сервером: \img{IconHelp} - вызов документации и диалога "о программе",
\img{IconShutdown} - выключение сервера. Закрытие панели управления сервера не приводит к выключению сервера -
он продолжает работать в фоновом режиме.
\n\tНа вкладке "Сети" осуществляется настройка кластерных сетей, членом которых является сервер.
Сервер может принадлежать одновременно к произвольному число сетей, но одновременно активной может быть не более одной.
Если сервер не является активным в рамках какой-либо сети, то его службы не функционируют.
Выход из сети приводит к разрыву всех соединений с другими узлами и всеми клиентами, останавливает службы.
Аналогично узлу, сеть имеет свойства имени и UUID. Для управления сетями существуют следующие команды:
\n\img{IconCreateNet} - создание новой сети. Пользователю будет предложено выбрать для сети имя и UUID.
Также потребуется указать порт, на котором будет функционировать данный сервер в этой сети.
Для UUID автоматически генерируется случайное значение, для порта - значение по умолчанию (10666).
Рекомендуется их оставить такими. Будет создана не активная сеть, состоящая из одного узла (текущего).
\n\img{IconJoinNet} - присоединение к сети. Для присоединения к действующей сети необходимо
указать IP адрес и порт любого активного узла этой сети. Также необходимо указать порт для данного узла.
Присоединение будет автоматически согласовано с другими узлами сети.
Для успешного присоединения к сети на целевом узле должно быть разрешено присоединение новых узлов, а сеть должна быть активной.
\n\img{IconForgetNet} - выход из сети. Исключает данный узел из выбранной сети. Если сеть активна и в ней более одного активного узла,
то факт выхода согласуется с другими узлами. Иначе другие узлы сети не будут оповещены о выходе.
При выходе из сети последнего узла, сеть прекращает существование.
\n\img{IconAutoconnectNet} - автоподключение. Объявляет выбранную сеть автоподключаемой (при запуске сервера).
Нажатие этой кнопки на сети с таким статусом снимет этот статус. Одновременно этот статус может иметь не более одной сети.
Это свойство является внутренним для сервера, не влияет на другие узлы, его изменение не приводит ни к каким мгновенным изменениям
сетевого состояния сервера.
\n\img{IconConnectNet} - подключиться. Делает выбранную сеть активной, запускает службы сервера.
\n\img{IconDisconnectNet} - отключиться. Деактивирует активную в данный момент сеть, разрывает все соединения и останавливает все службы.
\n\tНастройки, связанные с выбранной сетью, отображаются на вкладке "Узлы". В списке отображаются узлы сети и их статусы:
имя, подключён/отключён, состояние питания, состояние в рамках распределённых вычислений и прогресс вычислений.
Кнопки подключения и отключения дублируются на этой вкладке. Также имеются команды:
\n\img{IconAllowJoinNet} - разрешает или запрещает добавление новых узлов в эту сеть через данный узел.
Это свойство является внутренним для узла, равно "не разрешено" по умолчанию и сбрасывается в это состояние после каждого добавления.
\n\img{IconLeaveNet} - исключает выбранный узел из сети. Узел будет уведомлен об исключении, если он в данный момент активен.
\n\img{IconSelectNodes} - позволяет выбрать узлы сети, учавствующие в распределённых вычислениях.
\n\tТакже в контекстном меню (ПКМ) каждого узла доступны команды:
\n"Просмотреть службы узла" - выводит список всех участников кластера на данном узле (клиентов и сам сервер):
их имена службы, идентификаторы службы и адреса.
\n"Просмотреть свойства узла" - выводит информацию об аппаратном и программном обеспечении узла, его имя, адрес (сервера) и UUID.
\n"Прервать работу" - принудительно завершает распределённые вычисления. Следует использовать в случае сбоев.
\n"Управлять сервером" - позволяет дистанционно выключить сервер, выключить или перезагрузить ПК.
