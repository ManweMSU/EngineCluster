\hh{Работа с пакетами}\n\h{ecfxpack}\n\tСоздаёт пакет с несколькими вариантами программы.
Для пакета рекомендуется расширение .eipf. Синтаксис:
\n\code{ecfxpack <FILENAME> [-S] [-p <PROJECT> [-x | -s]]}
\n\code{ecfxpack <FILENAME> [-S] [-d <DIRECTORY> <WORDS> <ROOT>]}
\n S - бесшумный режим (без вывода текста в консоль).
\n p - создаёт пакет на основе проекта Engine Runtime.
\n x - в пакет будут включены версии в машинном коде, находящиеся в каталоге _build.
\n s - в пакет будет включена версия в исходном коде.
\n d - создаёт пакет на основе содержимого директорий. Можно указать несколько таких параметров. Не совместим с -p.
Для каждой директории следует указать набор слов, описывающих вариант программы (перечисленных одной строкой через -) и
имя "корневого" файла, который будет интерпретироваться как точка входа в программу или проект исходного кода соответственно.
\n\h{ecfxview}\n\tВыводит информацию о пакете. Синтаксис:
\n\code{ecfxview <FILENAME> [-S] [-x <WORDS> <DIRECTORY>] [-o]}
\n S - бесшумный режим (без вывода текста в консоль).
\n x - извлекает из пакета указанный вариант программы в указанную директорию.
\n o - разрешает перезаписывать уже существующие файлы при извлечении.
